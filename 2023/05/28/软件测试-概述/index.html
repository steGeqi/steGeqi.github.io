软件测试概述

软件测试的定义：

  • 定义一：使用人工或自动首都那来运行或测定某个系统的过程，其目的在于检验它是否满足规定的需求或是弄清预期结果与实际结果之间的差别
  • 定义二：根据软件开发的各个阶段的规格说明和程序的内部结构二精心设计的一批测试用例，并利用这些测试用例去执行程序，以发现软件故障的过程。（强调寻找故障是测试的目的）
  • 定义三：是一种软件质量保证活动，通过一些经济有效的方法发现软件中存在的缺陷，从而保证软件质量。

软件测试的目的

用尽可能少的时间和人力发现并改正软件中潜在的各种故障及缺陷，提高软件的质量

只能证明证伪不能证真

测试用例

测试用例 = 输入 + 输出

输入（数据） 、执行条件（操作步骤）、预期结果

软件测试的过程

软件测试与开发的关系

  • 软件测试贯穿于软件开发的整个生命周期
  • 六个阶段
    • 制定计划
      • 负责测试阶段的监控
    • 需求分析
      • 确定测试需求分析，指定系统测试计划
    • 软件设计
      • 指定集成测试计划和单元测试计划
    • 程序编写
      • 开发相应的测试代码或测试脚本
    • 软件测试
      • 实施测试，并提交相应的测试报告
    • 运行和维护

软件测试过程三大模型

V模型

特点：明确的表示了测试的不同级别，清晰的展示了软件测试于开发之间的关系

缺点：没有明确说明早期的测试，不能体现尽早和不断进行软件测试的原则

W模型

特点：形象的说明了测试与开发的并行关系，体现了测试贯穿于整个开发过程的思想

测试对象：程序 + 需求和设计阶段形成的文档

开发和测试仍然是串行活动，只有上一阶段完成后，才可以开始下一阶段的活动，不能支持迭代软件开发模型

H模型

特点：软件测试是独立的流程，贯穿于整个生命周期，与其他流程并发的进行。（强调测试是独立的，只要测试准备完成，就可以执行测试）

太过于模型化，重点在于理解其中的意义指导实际工作，而模型本身并无太多的可执行的指导意义。

X模型

探索性测试的应用，能够帮助有经验的测试人员在测试计划之外发现更多的软件错误。

测试环境的搭建

测试环境 = 硬件 + 软件 + 网络 + 数据准备 + 测试工具

课后习题：

1.软件测试分为哪几个测试阶段？各阶段的主要测试依据和测试目的分别是什么？

分为五个阶段：

​ 单元测试—以模块的详细设计说明书和源代码为依据，检测程序模块内部在逻辑上和功能上的故障。

​ 集成测试—以系统结构图为依据检测与接口有关的故障。

​ 确认测试—以需求规格说明书为依据，检测软件与所指定的需求的一致性。

​ 系统测试—以需求规格说明书为依据，在接近日常运行环境下进行综合测试，更注重性能的检测。

​ 验收测试—以最终用户的当前需求为依据，检测最终产品与之是否一致。

2.为什么说软件测试是软件开发中不可缺少的重要一环，但不是软件质量保证的安全网？

测试的工作量占到项目开发的40%以上，软件测试是对软件需求分析、设计规格说明、编码的最终复审，是软件质量保证的关键步骤，所以是开发中不可缺少的一环。不是安全网是因为测试只能发现错误，不能保证没有错误。

课堂题目

  1. 测试应该从分析需求阶段开始切入。

  2. 软件测试是采用测试用例执行软件的活动。

  3. 自动化测试工具包括：静态测试工具，黑盒测试工具，测试管理工具等。

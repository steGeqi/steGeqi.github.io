黑盒测试基本原理

黑盒测试是将被测软件看作一个打不开的黑盒，主要根据功能需求设计测试用例，进行测试。

从用户的观点出发的测试，其目的是尽可能发现软件的外部行为错误。

测试人员使用的唯一信息是：软件的规格说明。

优势：

  • 简单有效
  • 与软件的具体实施无关，如果软件实现发生了变化，测试用例依然可以使用
  • 设计黑盒测试用例可以和软件实现同时进行，因此可以压缩项目总的开发时间
  • 对测试人员的技术要求相对较低。

关于测试用例

测试用例尽量做到 针对性强、覆盖全面、无冗余、方法简单、有利于调试

常用方法

  1. 等价类划分法
  2. 边界值分析法（错误推测法）
  3. 决策表法、因果法
  4. 基于正交表的测试
  5. 基于场景的测试

等价类划分法

定义：

根据程序规格说明书对输入范围进行划分买吧所有可能的输入数据，即程序输入域划分为若干个互不相交的子集，称为等价类，然后从每个等价类中选取少数具有代表性的数据进行作为测试用例。

分为：有效等价类、无效等价类

常用等价类划分原则：

  • 按区间划分
  • 按数值划分
  • 按数值集合划分
  • 按限制条件或规则划分
  • 细分等价类

有效等价类测试用例设计覆盖标准：

弱覆盖：覆盖所有的有效等价类。（存在漏洞）

强覆盖：覆盖所有有效等价类的组合

等价测试类流程：

  1. 确定有几个输入条件
  2. 划分每个输入条件的等价类

表格抬头：输入条件、有效等价类、编号、无效等价类、编号

  1. 选择合适的覆盖标准
  2. 设计测试用例

测试用例编号、输入数据、预期输出结果、覆盖等价类

边界值分析法

  • 长期的测试工作经验表明，在输入域的边界或边界附近，常常会发现大量缺陷。
  • 是最基本、最简单的黑盒测试方法，具有很强的发现故障的能力。
  • 可作为等价类测试的补充。

边界值分析法流程：

  1. 确定有几个输入条件
  2. 确定每个输入条件的边界点
  3. 划定边界领域
  4. 穷尽法选择测试数据
  5. 结合测试数据设计测试用例
